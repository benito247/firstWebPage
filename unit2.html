Describing access issues that need taking into account describe issues that people with disabilities may face when using webpages and how we can overcome them

Technology is always changing finding more ways to improve itself a website is an online marketing tool businesses who invest time and energy into making there website user friendly will gain more customers because Technology is going to continue to find ways to overcome impossibilities leaving developers little room to not have tools to make websites more user friendly[10].[11] If the user is unable to display an image a alternative text would be in the image place[12]. If the user was to watch a video and is deaf or hard of hearing, provide captions whether spoken or in written text to make watching a video accessible, subtitles must be given to understand a video in any language there is a difference between captions and subtitles to be user friendly. If the user using a website is visually impaired give a descriptive text properly explaining the content links, for example if you are pointing visitors to a page called "About Us": Try not to say "click here to read about our company". Instead say "To learn more about our company read About us"[13]. Take into account the user may be affected by colour blindness which can affect the user from understanding what is been displayed the most common to be red and green. People who have other disabilities can greatly benefit from colours, the developer should understand when creating a website to use an asterisk or question mark to distinguish blocks of content from one another using visual separation (such as white space or borders)[14].
